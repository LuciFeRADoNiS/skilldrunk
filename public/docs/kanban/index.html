Product Vision

Why build another Kanban?

A lightweight, mobile-first Kanban board built into the skilldrunk ecosystem. Replaces Notion boards. Key difference: AI-aware — Claude (Cowork/Code) can read, create, and update tasks via API. "Assigned to Claude" means Cowork/Code will handle it.

🎯

Ecosystem Native

Same stack, same auth, same domain. No more context-switching to Notion.

🤖

AI-First Design

Claude is a first-class team member. API endpoints for read/create/update. Activity log with AI badge.

📱

Mobile-First

Swipe to move cards, pull-to-refresh, bottom nav, PWA installable. Built for on-the-go.

⚡

Realtime

Supabase Realtime for live updates. No refresh needed when Claude moves a card.

Supabase Schema

kanban_boards

CREATE TABLE kanban_boards (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name        text NOT NULL,
  slug        text UNIQUE NOT NULL,
  template    text DEFAULT 'default',
  columns     jsonb DEFAULT '["Backlog","Todo","In Progress","Review","Done"]',
  owner_id    uuid,
  is_public   boolean DEFAULT false,
  created_at  timestamptz DEFAULT now()
);

kanban_cards

CREATE TABLE kanban_cards (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  board_id    uuid REFERENCES kanban_boards(id),
  title       text NOT NULL,
  description text,
  column_name text NOT NULL DEFAULT 'Backlog',
  position    int DEFAULT 0,
  assignee    text,         -- person name or 'claude'
  priority    text DEFAULT 'P2',  -- P0, P1, P2, P3
  due_date    date,
  labels      text[],
  checklist   jsonb,
  is_archived boolean DEFAULT false,
  created_by  text,
  created_at  timestamptz DEFAULT now(),
  updated_at  timestamptz DEFAULT now()
);

kanban_comments

CREATE TABLE kanban_comments (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  card_id     uuid REFERENCES kanban_cards(id),
  author      text NOT NULL,
  content     text NOT NULL,
  created_at  timestamptz DEFAULT now()
);

kanban_activity

CREATE TABLE kanban_activity (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  card_id     uuid REFERENCES kanban_cards(id),
  actor       text NOT NULL,
  action      text NOT NULL,  -- 'moved', 'assigned', 'commented', 'created', 'completed'
  details     jsonb,
  created_at  timestamptz DEFAULT now()
);

kanban_people

CREATE TABLE kanban_people (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name        text NOT NULL,
  avatar_url  text,
  role        text DEFAULT 'member',  -- admin, member, viewer, ai_agent
  is_ai       boolean DEFAULT false,
  email       text,
  created_at  timestamptz DEFAULT now()
);

API Endpoints

REST API for Claude/Cowork and external integrations. All endpoints require API key or Supabase auth.

BOARDS

GET /api/kanban/boards List all boards
GET /api/kanban/boards/:slug Get board with all cards
POST /api/kanban/boards Create new board

CARDS

POST /api/kanban/cards Create card
PATCH /api/kanban/cards/:id Update card (move, assign, etc.)
DEL /api/kanban/cards/:id Archive card

COMMENTS & ACTIVITY

POST /api/kanban/cards/:id/comments Add comment
GET /api/kanban/cards/:id/activity Get activity log

Example: Create Card (Claude)

POST /api/kanban/cards
Content-Type: application/json
Authorization: Bearer SUPABASE_SERVICE_KEY

{
  "board_id": "skilldrunk-dev",
  "title": "Fix ISR cache invalidation",
  "description": "Cache not clearing on content update...",
  "column_name": "Todo",
  "assignee": "claude",
  "priority": "P1",
  "labels": ["bug", "performance"],
  "created_by": "claude"
}

Example: Move Card

PATCH /api/kanban/cards/abc-123
{
  "column_name": "Done",
  "assignee": "ozgur"
}

Claude / AI Integration

AI as Team Member

Claude isn't just a tool — it's a team member on the board. It can create, move, comment on, and complete cards. The activity log shows Claude's actions with an AI badge.

Capabilities

  • Create cards via Supabase API — when user says "bunu yapilacaklara ekle"
  • Move cards between columns (update column_name)
  • Add comments with progress updates or questions
  • Activity log — Claude shows as "Claude" with AI badge
  • Cowork dispatch — "create kanban card" action from natural language
  • Scheduled summary — daily kanban digest via Telegram/iMessage
COWORK FLOW

User says "bunu ekle" → Cowork parses intent → POST /api/kanban/cards → Card appears on board
DAILY SUMMARY FLOW

Cron (09:00) → Query active cards → Claude summarizes → Telegram / iMessage

Design System

  • Dark theme — consistent with skilldrunk ecosystem (#0d1117 base)
  • Gold accent (#e8b86d) for priority items and important actions
  • Smooth drag-drop — spring animations via @dnd-kit
  • Card preview on hover — description, checklist progress, assignee avatar

  • Typography — system font stack (-apple-system, BlinkMacSystemFont)
