🚀 Quick Start
Get GStack running in 30 seconds. Paste the install command into Claude Code.

1 Install GStack — open Claude Code and paste:

git clone --single-branch --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack cd ~/.claude/skills/gstack && ./setup

2 Try it out — run your first command:

/office-hours → describe what you're building /plan-ceo-review → challenge your plan /review → review your branch /qa → test your staging URL /ship → push and open PR

3 Team mode (optional) — share with teammates:

(cd ~/.claude/skills/gstack && ./setup --team) && \ ~/.claude/skills/gstack/bin/gstack-team-init required && \ git add .claude/ CLAUDE.md && git commit -m "require gstack for AI-assisted work"

🔗 Integration with Our Ecosystem
How GStack works alongside skilldrunk.com's existing infrastructure.

🌐 skilldrunk.com

GStack skills complement our AI Skills Marketplace. Use /office-hours before building new skills, /review before publishing, /qa to test skill endpoints.

🗃️ Supabase + RLS

/cso runs security audits including RLS policy validation. /plan-eng-review catches missing RLS on new tables. /investigate debugs Supabase edge function failures.

📅 Scheduled Tasks

Combine with Cowork scheduled tasks: nightly /health checks, weekly /retro reports, automated /canary monitoring after deploys.

🧠 Obsidian Vault

Feed meeting notes from Personal Brain into /office-hours for context. /document-generate creates Diataxis docs that sync to the vault.

🚀 Vercel Deploys

/ship opens PRs, /land-and-deploy merges and waits for Vercel CI, /canary monitors production health post-deploy.

🤖 Atlas Bot + Hermes

Use /review and /cso before deploying bot updates. /benchmark tracks response times. /investigate for systematic debugging of bot issues.

⚡ Before / After
What changes when you go from generic AI prompting to GStack's specialist agents.

❌ Before GStack

  • Single prompt, single persona
  • No structured review process
  • Manual QA in browser
  • Copy-paste PR descriptions
  • No security audit pipeline
  • Design review = visual inspection
  • No cross-session memory
  • Ship when it "looks right"

✅ After GStack

  • 23 specialist agents, each with a role
  • /review catches bugs CI misses
  • /qa opens real Chromium, clicks through flows
  • /ship runs tests, audits coverage, opens PR
  • /cso runs OWASP + STRIDE threat models
  • /design-review rates dimensions 0-10, fixes issues
  • /context-save + /context-restore across sessions
  • Think → Plan → Build → Review → Test → Ship → Reflect
