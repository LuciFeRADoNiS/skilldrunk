1. Problem Statement

Retired and aging artists possess valuable bodies of work — paintings, prints, mixed-media pieces — but have no practical way to showcase or sell them online. The core barriers:

  • Technical complexity: Platforms like Etsy, Shopify, or even Instagram require skills these artists don't have — SEO, product photography specs, pricing strategy, listing optimization.
  • Lost narratives: Each artwork carries a story, a technique, a lifetime of context. Generic marketplaces reduce art to a product card with a price tag.
  • Age & accessibility: Small text, cluttered interfaces, multi-step flows are hostile to users in the 60+ age group who often use tablets or phones with larger text settings.
  • Cultural gap: Turkish artists especially lack localized platforms. Etsy is English-first; Instagram is ephemeral; personal websites require ongoing maintenance.

2. Vision

A gallery-style platform where an artist uploads one photo and writes a few sentences — and AI does the rest. The platform generates a professional listing with description, story narrative, SEO tags, and pricing suggestion. The result is a beautiful, permanent online gallery that celebrates the story behind each piece, not just the transaction.

Think: mini Etsy meets artist portfolio, powered by AI, designed for simplicity.

3. AI-Powered Listing Flow

📷 Artist uploads photo
→
🔍 Claude Vision analyzes artwork
→
✍️ Artist writes 2-3 sentences
→
🤖 AI generates full listing
→
✅ Review & publish

What AI generates:

  • ✓ Professional title (if not provided by artist)
  • ✓ Full description (200-300 words) — technique, medium, composition analysis
  • ✓ Story narrative — connecting the artist's words to a compelling story
  • ✓ Style & technique tags — for search and discovery
  • ✓ Suggested price range — based on medium, size, artist profile
  • ✓ SEO-optimized metadata — for Google discoverability

4. Target Users

Primary: Retired Artists

60+ age group. Retired professors, self-taught artisans, hobbyist creators. Have years of work but zero online presence. Need extreme simplicity.

Secondary: Art Lovers

Collectors, gift buyers, people who value story-driven art. Want to discover unique pieces with authentic narratives, not mass-produced items.

Tertiary: Young Artists

Emerging artists without e-commerce skills or budget for a personal website. Need a zero-friction way to build a portfolio and reach buyers.

5. Unique Value Proposition

vs. Etsy

Focused on story, not transactions. No listing fees, no SEO optimization burden. AI does the heavy lifting.

vs. Instagram

Permanent gallery, not ephemeral feed. Structured artwork pages with full narratives. Discoverable by search engines.

vs. Personal Website

Zero technical skill needed. No hosting, no domain management, no updates. Just upload and write.

vs. Saatchi Art

Localized (Turkish-first). Accessible for non-English speakers. Warm, personal tone instead of corporate gallery feel.

8. Database Schema (Draft)

-- Artists
CREATE TABLE nb_artists (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name        TEXT NOT NULL,
  slug        TEXT UNIQUE NOT NULL,
  photo_url   TEXT,
  bio         TEXT,                    -- artist's own words (short)
  bio_full    TEXT,                    -- AI-expanded biography
  location    TEXT,
  techniques  TEXT[],                  -- ['serigrafi', 'gravür', 'karışık teknik']
  education   TEXT,
  active      BOOLEAN DEFAULT true,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Artworks
CREATE TABLE nb_artworks (
  id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  artist_id       UUID REFERENCES nb_artists(id),
  title           TEXT NOT NULL,
  slug            TEXT UNIQUE NOT NULL,
  image_url       TEXT NOT NULL,
  image_urls      TEXT[],              -- additional images
  artist_words    TEXT,                -- artist's 2-3 sentences
  ai_description  TEXT,                -- AI-generated description
  ai_story        TEXT,                -- AI-generated narrative
  ai_tags         TEXT[],              -- AI-generated style tags
  ai_analysis     JSONB,              -- Claude Vision analysis result
  medium          TEXT,                -- 'karışık teknik', 'serigrafi', etc.
  dimensions      TEXT,                -- '50x70 cm'
  year            INTEGER,
  price           DECIMAL(10,2),       -- optional
  price_suggested DECIMAL(10,2),       -- AI-suggested price
  currency        TEXT DEFAULT 'TRY',
  for_sale        BOOLEAN DEFAULT false,
  status          TEXT DEFAULT 'published' CHECK (status IN ('draft','published','sold','archived')),
  view_count      INTEGER DEFAULT 0,
  created_at      TIMESTAMPTZ DEFAULT now()
);

-- Comments (no auth required)
CREATE TABLE nb_comments (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  artwork_id  UUID REFERENCES nb_artworks(id),
  author_name TEXT NOT NULL,
  body        TEXT NOT NULL,
  approved    BOOLEAN DEFAULT true,   -- basic moderation
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Collections / Categories
CREATE TABLE nb_collections (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title       TEXT NOT NULL,
  slug        TEXT UNIQUE NOT NULL,
  description TEXT,
  cover_url   TEXT,
  sort_order  INTEGER DEFAULT 0
);

CREATE TABLE nb_artwork_collections (
  artwork_id    UUID REFERENCES nb_artworks(id),
  collection_id UUID REFERENCES nb_collections(id),
  PRIMARY KEY (artwork_id, collection_id)
);

-- RLS: public read, authenticated write for artist's own data
ALTER TABLE nb_artists ENABLE ROW LEVEL SECURITY;
ALTER TABLE nb_artworks ENABLE ROW LEVEL SECURITY;
ALTER TABLE nb_comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE nb_collections ENABLE ROW LEVEL SECURITY;

-- Public read policies
CREATE POLICY "Public read artists" ON nb_artists FOR SELECT USING (true);
CREATE POLICY "Public read artworks" ON nb_artworks FOR SELECT USING (status = 'published');
CREATE POLICY "Public read comments" ON nb_comments FOR SELECT USING (approved = true);
CREATE POLICY "Public read collections" ON nb_collections FOR SELECT USING (true);
CREATE POLICY "Public insert comments" ON nb_comments FOR INSERT WITH CHECK (true);

10. Design Philosophy

  • Gallery aesthetics: Warm, cream/off-white backgrounds with serif fonts for titles. Art is the hero — large images, generous whitespace.
  • Story-first layout: Artist stories prominently featured alongside artwork. Every piece has a narrative, not just a product card.
  • Accessibility: Large text (min 16px body), simple navigation, high contrast. Designed for users who may have vision limitations.
  • Mobile-first: Many target users (60+) primarily use tablets and phones. Touch-friendly, large tap targets, no hover-dependent interactions.
  • Minimal chrome: No sidebars, no widget clutter, no pop-ups. Clean, calm, gallery-like experience.
  • Cultural sensitivity: Turkish language defaults, Turkish art terminology, Anatolian visual references in UI accents.

11. AI Classification Prompt (Draft)

System: You are an art curator and listing specialist for a Turkish
artist platform. You analyze artwork images and artist notes to create
compelling, accurate, and culturally sensitive art listings.

When given an artwork image and the artist's brief notes:

1. ANALYZE the image:
   - Medium/technique (yağlıboya, akrilik, serigrafi, gravür, karışık teknik, etc.)
   - Color palette and dominant tones
   - Composition and visual structure
   - Mood and emotional resonance
   - Cultural/mythological references if present

2. GENERATE:
   - Title (if not provided): evocative, Turkish, reflecting the artwork's essence
   - Description (200-300 words): technical analysis + emotional interpretation
   - Story narrative (150-200 words): connecting artist's words to a broader narrative
   - Tags: 5-8 style/technique/theme tags in Turkish
   - Price suggestion: based on medium, technique complexity, size

3. TONE:
   - Warm, respectful, culturally aware
   - Avoid art-world jargon — accessible to non-specialist readers
   - Honor the artist's intent — amplify, don't replace their voice
   - Turkish language, with option to generate English translation

13. Success Criteria

  • MVP launch: Nevra Bozok profile + 10 artworks live with AI-generated listings
  • Usability: Artist can upload a new artwork in under 2 minutes (photo + 2-3 sentences → published)
  • AI quality: 80%+ of AI-generated descriptions accepted without major edits
  • Engagement: Average 2+ minutes on artwork pages (story reading)
  • Accessibility: Lighthouse accessibility score > 90
  • Multi-artist: Platform supports 3+ artists by Phase 2 launch
