Core UX — Swipe-to-Like

Design Philosophy

Mobile-first, Tinder-style card interface. One quote fills the viewport. Swipe right to like, left to skip. Thumb-friendly, zero-friction engagement. Desktop: arrow keys + click fallback.

👆

Swipe Right = Like

Touch gesture with spring animation. Card flies off-screen with a heart pulse effect. Like count increments in real-time.

👈

Swipe Left = Skip

Card dismisses left with a subtle fade. No negative action stored — just advances to the next quote.

❤️

Like Counter

Anonymous, stored in Supabase. Each quote shows total likes. No auth required — session-based dedup.

💬

Comments

Name + comment form below each quote. No auth — just name field. Chronological display, newest first.

💡

Suggest a Quote

Floating action button opens a modal form: quote text + author + submitter name. Stored for admin review.

🎴

Card Stack UI

3-card stack visible: current card on top, next two peeking behind with parallax depth. Preloads next 5 quotes.

Load page → Fetch batch (10 quotes) → Display card stack → Swipe right / left → Track + animate → Next card → Batch exhausted? Fetch more

Social Sharing

Every quote has a persistent share bar. Platform-specific formatting ensures the shared content looks native on each channel.

📱

WhatsApp

Clean text: quote + attribution + link. Uses wa.me deep link.

📸

Instagram

Canvas API generates 1080×1080 dark-bg PNG. Download button for IG Stories.

🐦

Twitter / X

Pre-formatted tweet with quote, em-dash attribution, and short link.

📋

Copy to Clipboard

One-tap copy with “Copied!” toast animation. Includes attribution.

🔗

Native Share

Web Share API for mobile share sheet. Falls back to copy-link on desktop.

Instagram Story Generator — Technical Detail

Uses HTML5 Canvas API to render a 1080×1080px image:

  • Dark gradient background (#0d1117 → #161b22)
  • Quote text: white, centered, auto-wrapped, size scaled to fit
  • Attribution line: accent color, smaller font, below quote
  • Subtle quotes.skilldrunk.com watermark at bottom
  • Export as PNG via canvas.toBlob() → download trigger

Personalization (Cookie-Based)

No authentication required. All personalization uses localStorage + a session cookie for cross-tab sync.

⭐

Save Favorites

Tap the star icon to save a quote to your local collection. Persists via localStorage. Sync indicator on load.

📚

My Collection

Dedicated page showing all saved quotes. Filterable, sortable by date saved. Export as text or image grid.

🕐

View History

Track which quotes you’ve already seen. “Show unseen only” toggle in settings. History stored locally.

📧

Email Subscription CTA

After 5+ favorites saved: banner appears — “Get a daily quote in your inbox.” Email form → email_subscribers table.

User saves 5 quotes → CTA banner slides in → Email input + subscribe → Supabase insert → Daily cron sends email

Analytics

Every interaction is tracked for data-driven iteration. Anonymous session fingerprinting (no PII). All data flows to Supabase for real-time dashboards.

Aggregate Metrics (Daily Cron)

  • Most Liked — Top 10 quotes by like count (24h / 7d / all-time)
  • Most Shared — Top 10 by share count, broken down by platform
  • Trending — Velocity formula: (likes_24h * 2 + shares_24h * 3) / hours_since_created
  • Engagement Rate — (likes + shares + comments) / views per quote
  • Session Depth — Average quotes viewed per session, time on page

Supabase Schema

All tables use RLS. Public read on aggregates, session-validated writes. Service role for cron jobs only.

-- Quote engagement tables
-- Prefix: none (extends existing qt_quotes table)

CREATE TABLE quote_likes (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  quote_id int REFERENCES quotes(id),
  session_id text NOT NULL,
  created_at timestamptz DEFAULT now()
);
-- Unique constraint: one like per session per quote
CREATE UNIQUE INDEX idx_quote_likes_unique
  ON quote_likes(quote_id, session_id);

CREATE TABLE quote_comments (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  quote_id int REFERENCES quotes(id),
  author_name text NOT NULL CHECK (char_length(author_name) BETWEEN 1 AND 50),
  comment_text text NOT NULL CHECK (char_length(comment_text) BETWEEN 1 AND 500),
  created_at timestamptz DEFAULT now()
);

CREATE TABLE quote_shares (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  quote_id int REFERENCES quotes(id),
  platform text NOT NULL CHECK (platform IN
    ('whatsapp','instagram','twitter','copy','native')),
  session_id text NOT NULL,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE quote_views (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  quote_id int REFERENCES quotes(id),
  session_id text NOT NULL,
  view_duration_ms int DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE email_subscribers (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  email text UNIQUE NOT NULL,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- RLS policies
ALTER TABLE quote_likes ENABLE ROW LEVEL SECURITY;
ALTER TABLE quote_comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE quote_shares ENABLE ROW LEVEL SECURITY;
ALTER TABLE quote_views ENABLE ROW LEVEL SECURITY;
ALTER TABLE email_subscribers ENABLE ROW LEVEL SECURITY;

-- Public insert (anon key), admin read-all
CREATE POLICY "anon_insert_likes" ON quote_likes
  FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_comments" ON quote_comments
  FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_shares" ON quote_shares
  FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_views" ON quote_views
  FOR INSERT WITH CHECK (true);
CREATE POLICY "anon_insert_subscribers" ON email_subscribers
  FOR INSERT WITH CHECK (true);

Ecosystem Access & Tracking

Temporary Access Links

Generate time-limited tokens for sharing specific pages with external users. Track their behavior after landing.

  • Token format: UUID v4, stored in access_tokens table
  • URL: quotes.skilldrunk.com/?token=abc123
  • Configurable TTL: 1 hour, 24 hours, 7 days, or custom
  • Each token scoped to a specific target URL or quote ID

Referred User Behavior Tracking

  • On token-based landing: create a tracked session linked to the token
  • Track: pages visited, quotes liked/shared, time spent, scroll depth
  • Dashboard view in admin: “Who clicked your link and what did they do?”
CREATE TABLE access_tokens (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  token text UNIQUE NOT NULL DEFAULT gen_random_uuid()::text,
  target_url text NOT NULL,
  expires_at timestamptz NOT NULL,
  created_by text, -- admin email or 'system'
  views int DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE access_tokens ENABLE ROW LEVEL SECURITY;
CREATE POLICY "public_read_valid_tokens" ON access_tokens
  FOR SELECT USING (expires_at > now());
