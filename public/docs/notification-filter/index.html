1. Problem Statement

Too many notifications from too many apps fragment focus and create constant context-switching. Current state: iOS notifications from ~30+ apps, Mac notifications from ~20 apps, Telegram/WhatsApp/iMessage/Slack/Email all competing for attention simultaneously.

There is no single filtered channel. The user must manually triage hundreds of daily notifications, missing important ones while being interrupted by noise.

2. Vision

All notifications (iOS, Mac, web services) flow through a central AI filter. Claude classifies each notification into one of four categories:

CategoryBehaviorExamples
URGENTImmediate delivery via Telegram pushServer down alert, meeting in 5min, money transfer, boss DM
IMPORTANTBatched every 2 hoursPR review request, client email, scheduled event reminder
CAN-WAITDaily digest (morning + evening)Newsletter, social media mention, app update, shipping notification
NOISELogged but never pushedMarketing email, game notification, promotional push

Full history is searchable via skimsoulfat.com dashboard and /bildirim Atlas command.

3. Architecture Options

Option A: iOS App + Mac App (Native)
Best UX. iOS Notification Service Extension intercepts push notifications before display. Mac app uses UserNotifications framework. Hardest to build — requires Apple Developer account, Swift expertise, App Store review.
Option B: iOS Shortcuts + Focus Mode Automation (Quick Win)
Leverage existing iOS Shortcuts to forward notifications to a webhook. Limited by Shortcuts capabilities — can't intercept all notification types. Focus Mode profiles for time-based filtering. Good as Phase 0 experiment.
Option C: Proxy Notification Service (Server-Side) ★ RECOMMENDED FOR MVP
Mac AppleScript captures notifications → sends to Supabase → Claude API classifies → important ones forwarded to Telegram via Atlas bot. Cross-platform, leverages existing infrastructure. Can run as Cowork scheduled task.

4. Technical Architecture (MVP — Option C)

Mac Notifications → AppleScript Capture → Supabase (raw_notifications) → Claude API Classifier → Atlas Telegram Forward

4.1 Notification Capture (Mac)

Use macOS Notification Center database + AppleScript to capture notifications. Two approaches:

-- Approach 1: Read notification center SQLite DB
-- Location: ~/Library/Group Containers/group.com.apple.usernoted/db2/db
-- SELECT * FROM record WHERE delivered_date > ?

-- Approach 2: osascript notification listener (Cowork scheduled task)
-- Poll every 60 seconds, capture new notifications
-- Store: app_name, title, body, timestamp, bundle_id

Cowork scheduled task runs every 60 seconds, reads new notifications since last poll, inserts into Supabase.

4.2 Supabase Schema

-- Table: notif_raw (all captured notifications)
CREATE TABLE notif_raw (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  source      TEXT NOT NULL,           -- 'mac', 'ios', 'web'
  app_name    TEXT NOT NULL,           -- 'Mail', 'Slack', 'WhatsApp'
  bundle_id   TEXT,                    -- 'com.apple.mail'
  title       TEXT,
  body        TEXT,
  captured_at TIMESTAMPTZ DEFAULT now(),
  classified  BOOLEAN DEFAULT false
);

-- Table: notif_classified (after AI classification)
CREATE TABLE notif_classified (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  raw_id      UUID REFERENCES notif_raw(id),
  category    TEXT NOT NULL CHECK (category IN ('urgent','important','can_wait','noise')),
  reason      TEXT,                    -- AI's reasoning
  forwarded   BOOLEAN DEFAULT false,
  forwarded_at TIMESTAMPTZ,
  batch_id    TEXT,                    -- for grouped delivery
  classified_at TIMESTAMPTZ DEFAULT now()
);

-- Table: notif_rules (user-defined overrides)
CREATE TABLE notif_rules (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  app_name    TEXT,
  bundle_id   TEXT,
  keyword     TEXT,                    -- title/body contains
  sender      TEXT,                    -- from specific sender
  override_to TEXT NOT NULL CHECK (override_to IN ('urgent','important','can_wait','noise','block')),
  active      BOOLEAN DEFAULT true,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Index for fast polling
CREATE INDEX idx_notif_raw_unclassified ON notif_raw(classified, captured_at)
  WHERE classified = false;
CREATE INDEX idx_notif_classified_pending ON notif_classified(forwarded, category)
  WHERE forwarded = false;

4.3 AI Classification

Claude Haiku for speed and cost. Classification prompt:

System: You are a notification triage assistant for Özgür GÜR, CTO.
Classify each notification into exactly one category:
- urgent: requires immediate action (server alerts, meeting in <15min,
  direct messages from key people, financial transactions)
- important: should be seen within 2 hours (work emails, PR reviews,
  calendar events, client messages)
- can_wait: daily digest material (newsletters, social media, shipping,
  app updates, non-urgent group messages)
- noise: log only (marketing, promotions, games, spam)

Context: User works at EGV Group (ENCO, MoveTech). Key contacts include
team leads and board members. High priority apps: Telegram, Slack, Mail,
Calendar. Low priority: social media, marketing.

User rules (override AI judgment):
{user_rules_from_notif_rules_table}

Respond with JSON: {"category": "...", "reason": "..."}

Estimated cost: ~$0.01-0.02/day (Haiku, ~200 notifications × ~100 tokens each).

4.4 Delivery via Atlas

Atlas bot receives classified notifications via Supabase trigger or Cowork scheduled task:

  • Urgent: Immediate Telegram push with 🔴 URGENT prefix
  • Important: Batched every 2h, sent as a grouped message with 🟠 IMPORTANT header
  • Can-wait: Morning (07:00) and evening (19:00) digest
  • Noise: Never forwarded, searchable via /bildirim noise

Atlas commands:

/bildirim              — Son 24h özet (kaç urgent/important/wait/noise)
/bildirim urgent       — Son urgent bildirimleri listele
/bildirim ara <query>  — Full-text search tüm bildirimlerde
/bildirim kural ekle   — Yeni override kuralı ekle
/bildirim sessiz <app> — Bir app'i noise'a override et

7. Success Criteria

  • Zero missed urgent notifications (false negative rate < 1%)
  • Noise reduction: >60% of notifications classified as can_wait or noise
  • Response to urgent: <2 minutes from notification to Telegram delivery
  • User satisfaction: fewer context switches per day, measurable via skimsoulfat dashboard
  • Cost: under $15/month total
